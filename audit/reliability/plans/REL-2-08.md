Approach:

1. **Label cardinality budget** (canonical):
   | Label key   | Cardinality bound                | Source of bound          |
   |-------------|----------------------------------|--------------------------|
   | `listener`  | bounded by `config.listeners.len()` в‰¤ 32 | static config            |
   | `backend`   | bounded by `ОЈ pool.max_per_listener` в‰¤ 256 | static config           |
   | `route`     | bounded by `config.routes.len()` в‰¤ 64 | static config            |
   | `status_class` | 5 (`1xx`,`2xx`,`3xx`,`4xx`,`5xx`)  | enum                     |
   | `version`   | 4 (`HTTP/1.0`,`HTTP/1.1`,`HTTP/2`,`HTTP/3`) | enum                     |
   | `kind`/`action`/`result` | в‰¤ 8 (per family-specific enum)  | per family               |

   Worst-case product (32Г—256Г—64Г—5Г—4 в‰€ 5M) is too large; we **don't
   multiply all labels together**. Family-by-family rule in `LabelBudget`:
   - `http_requests_total{listener,route,version,status_class}` вЂ” 32Г—64Г—4Г—5 = 41 k. **No `backend`** on request metrics.
   - `http_request_duration_seconds{listener,route,version}` вЂ” 32Г—64Г—4 = 8 k (Histogram, no status_class).
   - `backend_requests_total{listener,backend,status_class}` вЂ” 32Г—256Г—5 = 41 k.
   - `backend_request_duration_seconds{listener,backend}` вЂ” 32Г—256 = 8 k.
   - `connections_inflight{listener}` вЂ” Gauge, 32.

2. **Per-request placement**: today RED increments at connection
   scope (`main.rs:1186`), double-counting keep-alive reuse. New
   `RedRecorder::record_request(listener, route, version, status,
   dur)` API called by sec/code at L7 entry points, once per request.

3. **Startup enforcement**: `LabelBudget::register(&cfg)` computes
   theoretical max series from config; refuses to start if any
   family exceeds 100 k. Hard rail against misconfig prometheus-OOM.

4. Drop the conn-scope counter at `main.rs:1186` once wired (code).

5. `connections_inflight{listener}` Gauge via `Arc<AtomicI64>` per
   listener вЂ” registered here; saturation signal for REL-2-09.

Risk / blast radius:
Re-registering metric families is a one-way trip вЂ” old series
disappear and new series appear at the next scrape. Operators with
dashboards on the old names need migration; covered by REL-2-01
doc-sweep + a `METRICS.md` migration table. Worst-case cardinality
escape: bug that accepts unbounded `route` strings в†’ prometheus OOM.
Mitigation: route label is sourced from the config's route name,
not request headers; impossible to extend at runtime.
