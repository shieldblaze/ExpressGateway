Approach:
This plan owns the **ordering contract** + the **integration proof**.
Cancellation-token plumbing (CODE-2-03) and GOAWAY/CONNECTION_CLOSE
frame emission (PROTO-2-11) are owned by code and proto respectively
and are dependencies, not edits made here.

Drain sequence (the ordering rel guarantees):

```
T+0 ms   SIGTERM received in async_main
         → set DRAIN_STATE = Draining (Arc<AtomicU8>) and:
         → /readyz starts returning 503 (REL-2-04 listens for state)
         → emit shutdown_started_at_seconds gauge

T+0 ms   Wait `[runtime].readiness_settle_ms` (default 1000 ms) so the
         external LB observes the 503 and stops sending new requests.

T+1000   For each ListenerState (TCP/H1/H1s/H2/QUIC):
         → cancel listener.accept_token (code-owned CancellationToken,
           CODE-2-03) — stops accept loop.
         → emit listener_accepting{listener} = 0.

T+1000   Per-protocol graceful close (proto-owned, PROTO-2-11):
         → H1: response builder appends `Connection: close`.
         → H2: hyper::server::conn::http2::Connection::graceful_shutdown()
           emits GOAWAY with last_stream_id.
         → H3/QUIC: quinn::Connection::close(QUIC_NO_ERROR, b"shutdown")
           emits CONNECTION_CLOSE.
         → Plain TCP: half-close client→backend; pump backend→client.
         → XDP: drop XdpLoader LAST, after userspace inserter flushes
           (per EBPF-2-05 amendment in rel cross-review §1).

T+0..D   Wait on per-listener `JoinSet::join_all()` with deadline =
         shutdown_started_at + drain_timeout_ms (default 10 000 ms,
         configurable via `[runtime].drain_timeout_ms`).

T+D      Hard cap: for each still-live task in any JoinSet, call
         `JoinHandle::abort()`. Increment
         shutdown_aborted_connections_total{listener,protocol}.

         Record shutdown_drain_seconds histogram (end - start).

         Process exit 0.
```

Config addition in `[runtime]`:
```toml
[runtime]
drain_timeout_ms = 10000        # hard cap on graceful drain
readiness_settle_ms = 1000      # delay between /readyz=503 and accept-stop
```

Metrics emitted (registered in `lb-observability`):
- `shutdown_drain_seconds` (Histogram, buckets 0.1, 0.5, 1, 2, 5, 10, 20).
- `shutdown_aborted_connections_total{listener,protocol}` (Counter).
- `shutdown_started_at_seconds` (Gauge, unix-ts; set once).
- `listener_accepting{listener}` (Gauge, 0/1).

Proof:
New test `tests/reload_zero_drop.rs::test_sigterm_drains_h2_with_goaway`.
- Spawn binary with a 200 MB streaming H2 endpoint on port 0.
- Open one client streaming `body_size=200 MB`.
- Send SIGTERM. Assert in order:
  1. `/readyz` returns 503 within `readiness_settle_ms + 100 ms`.
  2. Client receives a GOAWAY frame (parse the wire via `h2`).
  3. Streamed body completes with no `RST_STREAM`, no TCP RST.
  4. Process exits within `drain_timeout_ms + 500 ms`.
  5. `shutdown_aborted_connections_total` == 0 for this run.
A second matrix variant `test_sigterm_drains_h1_and_quic` runs the
same assertion for H1 (Connection: close) and QUIC (CONNECTION_CLOSE).

Risk / blast radius:
The accept-loop change touches the hottest path in the binary; the
JoinSet refactor is owned by `code` (CODE-2-03). rel's contribution
is config + metrics + sequencing test; risk localised to drain code
path. Worst-case regression is a longer-than-necessary drain — caught
by the test's deadline assertion.
