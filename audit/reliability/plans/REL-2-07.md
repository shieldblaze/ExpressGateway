Approach:

1. **Helper API** in `lb-observability::trace_context`:
   ```rust
   pub fn extract_parent(headers: &HeaderMap) -> Option<SpanContext>;
   pub fn inject_into_request<T: HttpHeaders>(span: &Span, req: &mut T);
   pub fn make_request_span(
       listener: &str,
       method:   &str,
       target:   &str,
       version:  http::Version,
       parent:   Option<SpanContext>,
   ) -> tracing::Span;
   ```
   Span fields follow OTel semantic conventions:
   `http.request.method`, `url.path`, `network.protocol.version`,
   `server.address`, `client.address`. Status + bytes recorded on
   exit.

2. **Sampler**, configurable in `[observability.tracing.sampler]`:
   ```toml
   [observability.tracing.sampler]
   kind = "parent_based_traceidratio"     # | "always_on" | "always_off" | "traceidratio"
   ratio = 0.01                            # 1% root sampling
   ```
   Default: `parent_based_traceidratio` with ratio 0.01 â€” respects
   upstream sampling decisions, samples 1% of root spans.

3. **Exporter**, OTLP/gRPC by default:
   ```toml
   [observability.tracing.exporter]
   kind     = "otlp_grpc"
   endpoint = "http://localhost:4317"
   timeout_ms = 3000
   ```
   Endpoint configurable; bound to admin listener address space but
   does NOT share the admin TCP socket (OTel batches outbound).

4. **Header policy**: if request has valid `traceparent`, adopt as
   parent and forward unchanged; if absent, synthesize a root span
   and inject `traceparent` (unless `inject_when_absent = false`).
   `tracestate` byte-for-byte pass-through after length check.

5. **JSON log coupling** (REL-2-06): formatter set with
   `with_current_span(true)`; `trace_id`/`span_id` appear
   automatically on every log line emitted inside the span.

6. **Upstream child spans** `lb.upstream_dial` and
   `lb.upstream_first_byte` opened by code in the connect path
   (CODE-2-09). rel defines the names/attrs.

3. `test_log_line_carries_trace_id`:
   send 1 request; capture stdout JSON line for that request;

4. `test_sampler_ratio`:
   `ratio=0.1`; send 10 000 requests with no parent context; assert
   exported span count is within 9% of 1000 (statistical bound).

Risk / blast radius:
OTLP exporter is non-blocking (batch processor with bounded queue);
drops on overflow are surfaced via a `traces_dropped_total` counter
we'll register in the exporter's runtime. Worst case: misconfigured
endpoint silently drops traces â€” caught by the same counter +
alert. Wire-format risk: `traceparent` parsing is RFC-strict; pass-
through always preserves the original even when LB parsing fails
(`extract_parent` returns `None` but the header is still forwarded).
