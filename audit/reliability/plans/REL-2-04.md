# Plan for REL-2-04 — `/livez` + `/readyz` + `/startupz` split
Finding-ref:     REL-2-04 (high, Open)
Files touched:
- `crates/lb-observability/src/admin_http.rs` (add three new routes;
  current file 151 lines — extend the match arm at line ~55 that
  currently serves `/healthz`)
- `crates/lb-observability/src/lib.rs` (re-export
  `ProbeState` enum + `ProbeRegistry` Arc shared with main)
- `crates/lb/src/main.rs` (wire ProbeRegistry into ListenerState
  bind/drain transitions)
- `tests/health_probes.rs` (new)

Approach:
Add a `ProbeRegistry` shared across the binary that tracks three
booleans + a per-listener accepting map:

```rust
pub struct ProbeRegistry {
    pub runtime_up:    AtomicBool,          // flipped true once in async_main
    pub startup_done:  AtomicBool,          // flipped true once all listeners bound
    pub drain_state:   AtomicU8,            // 0=live, 1=draining, 2=stopped
    pub listeners:     DashMap<String, AtomicBool>, // accepting flag per bind addr
}
```

Routes (added to `admin_http.rs`'s match arm; see existing `/healthz`
handler in the same file):

| Method | Path        | 200 condition                                                    | 503 body                          |
|--------|-------------|------------------------------------------------------------------|-----------------------------------|
| GET    | `/livez`    | `runtime_up == true`                                             | `{"status":"runtime_down"}`       |
| GET    | `/readyz`   | `runtime_up && startup_done && drain_state==0 && all listeners.accepting==true` | `{"status":"<draining\|listener_down\|booting>"}` |
| GET    | `/startupz` | `runtime_up && startup_done`                                     | `{"status":"booting"}`            |
| GET    | `/healthz`  | back-compat alias for `/livez` (kept for old consumers)          | as `/livez`                       |

All four respond with `Content-Type: application/json; charset=utf-8`.
200 body: `{"status":"ok"}`. 503 body: structured reason as in table.

Transitions (cross-reference REL-2-02):
- `runtime_up = true` at the top of `async_main`, before any spawn.
- `listeners[bind].accepting = true` after first successful
  `listener.accept()` returns on that listener.
- `startup_done = true` once all configured listeners have their
  accepting flag set AND DNS resolution for static backends completed.
- On SIGTERM (REL-2-02 step a): `drain_state = 1` → `/readyz` flips
  to 503 *before* anything else happens.
- Per-listener `accepting = false` set by the listener's drain task
  (REL-2-02 step b).

Loopback-bind assumption: admin listener is already loopback-only
(see `sec` SEC-2-06). No auth needed on these three routes
(documented in REL-2-01 doc-sweep).

Proof:
`tests/health_probes.rs`:
1. `test_startupz_blocks_until_listeners_bound`:
   spawn with a synthetic listener that delays bind by 1 s. Assert
   `/startupz` returns 503 for the first ~1 s, 200 thereafter.
2. `test_readyz_flips_503_on_sigterm`:
   spawn, wait ready. Send SIGTERM. Within 50 ms of signal delivery,
   `/readyz` returns 503 `{"status":"draining"}`. `/livez` still 200
   until process exit.
3. `test_readyz_503_when_listener_unbound`:
   misconfigure one of two listeners to fail bind. `/readyz` returns
   503 `{"status":"listener_down"}` while `/livez` returns 200.
4. `test_healthz_alias`:
   `/healthz` response body and status identical to `/livez` across
   all states.

Risk / blast radius:
admin_http.rs is small (151 lines) and rel-owned; the change is
additive on the route match arm. Negative risk: external monitoring
that polls `/healthz` will switch semantics-by-default from "the
process is up" to (intentionally) the same — kept identical via the
alias. K8s manifests will need updating but those are doc-only
(covered by REL-2-01 + DEPLOYMENT.md).
