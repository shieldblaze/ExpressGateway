# Plan for REL-2-03 — TLS cert rotation wired via `ArcSwap<TlsConfigBundle>` + SIGUSR1
Finding-ref:     REL-2-03 (critical, Open)
Files touched:
- `crates/lb-security/src/ticket.rs` (extend to expose
  `TlsConfigBundle` + `ArcSwapBundle` types; rel's owned file)
- `crates/lb-observability/src/lib.rs` (cert metrics)
- `crates/lb/src/main.rs` (signal handler + inotify watcher; coordinate
  with `code` — rel owns the signal-glue shape, code lands the edit)
- `tests/cert_rotation.rs` (new — `test_sigusr1_rotates_cert_no_drop`)

Approach:
Lead pre-approved direction is `ArcSwap` + rustls `ResolvesServerCert`.
`sec` co-owns cert-validation correctness (chain depth, key-match,
expiry); rel owns the rotation plumbing + metrics.

Design:

1. New module `crates/lb-security/src/ticket.rs::tls_bundle`:
   ```rust
   pub struct TlsConfigBundle {
       pub cert_chain: Vec<CertificateDer<'static>>,
       pub key:        PrivateKeyDer<'static>,
       pub not_after:  SystemTime,
       pub resolver:   Arc<dyn ResolvesServerCert>,
       loaded_at:      Instant,
   }
   pub type SharedTlsBundle = Arc<ArcSwap<TlsConfigBundle>>;
   ```
   Validation on load: `rustls::server::ConfigBuilder` smoke-build,
   chain depth ≤ 6, key-pair match via `SubjectPublicKeyInfo`
   comparison, `not_after > now + 24 h` (warn only; do not refuse).

2. `TlsConfigBundle::load_from_paths(cert: &Path, key: &Path) ->
   Result<Self, TlsBundleError>` — used at startup AND on reload.

3. Every TLS listener stops holding `Arc<TlsAcceptor>`; instead it
   holds `SharedTlsBundle` and constructs a `TlsAcceptor` per accept
   from `bundle.load().resolver.clone()`. (Cheap — resolver is an
   Arc-wrapped trait object.)

4. **Trigger** — two paths, both lead to `reload_tls()`:
   - `SIGUSR1`: explicit operator signal (preferred for runbook
     scripts). Handler installed in `async_main` alongside the
     SIGTERM handler.
   - `inotify` watch on the parent dir of cert+key (debounced 500 ms,
     `notify` crate). Triggers reload on `MODIFY`/`MOVED_TO` events.
   Both paths funnel through:
   ```rust
   async fn reload_tls(bundle: SharedTlsBundle, paths: TlsPaths) {
       match TlsConfigBundle::load_from_paths(&paths.cert, &paths.key) {
           Ok(new) => { bundle.store(Arc::new(new)); metric ok; }
           Err(e)  => { tracing::error!(?e); metric fail; }   // keep old
       }
   }
   ```

5. In-flight handshakes keep the snapshot they loaded; new handshakes
   pick up the swapped bundle on next `accept()`. No connection
   reset.

Metrics (in `lb-observability`):
- `tls_cert_reload_total{result}` — result ∈ {ok, validation_failed,
  io_error, key_mismatch}.
- `tls_cert_not_after_seconds{listener}` — unix-ts gauge.
- `tls_cert_loaded_at_seconds{listener}` — unix-ts gauge.
- `tls_cert_in_use_serial{listener}` — info-style gauge (label-only,
  value = 1) for grafana annotation.

Proof:
`tests/cert_rotation.rs::test_sigusr1_rotates_cert_no_drop`:
- Start binary with cert A (CN=A, valid 1 year).
- Open 32 long-lived TLS connections doing 1 KB/s pings.
- Replace cert files with cert B (CN=B); send SIGUSR1.
- Assertions:
  1. All 32 long-lived connections continue without RST or TLS close
     for the full 30 s observation window.
  2. A *new* connection opened post-signal sees CN=B in
     `ssl_handshake.server_certificates[0].subject`.
  3. `tls_cert_reload_total{result="ok"}` increments by 1.
  4. `tls_cert_not_after_seconds{listener=":443"}` reflects cert B.

Second test `test_invalid_cert_keeps_old_live`: replace key with
mismatched key; assert `result="key_mismatch"` and existing
connections + new connections still serve cert A.

Risk / blast radius:
The "TlsAcceptor per accept" allocation is a microsecond — measured
under benches in `bench/`. If it shows up in flame graphs, fallback
is `ResolvesServerCert` trait inside a single permanent `TlsAcceptor`
(cheaper but more code). Validation strictness may reject some
operator certs — start with depth ≤ 6 + key-match, defer SAN/EKU
checks to `sec`.
