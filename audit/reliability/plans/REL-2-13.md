Approach:

REL-2-13 is the **superset** sibling of REL-2-12. While REL-2-12
focuses on the CONNTRACK saturation alert, REL-2-13 exports every
field of the EBPF-2-08 `StatsSnapshot` that doesn't already live
on the CONNTRACK alert surface.

Families this plan registers:

| Metric | Type | Labels | Source field |
|---|---|---|---|
| `xdp_packets_total` | Counter | `action="forwarded"\|"dropped"\|"passed_to_kernel"\|"tx"\|"redirect"` | `packets[Action::*]` |
| `xdp_bytes_total` | Counter | `direction="rx"\|"tx"` | `bytes[Direction::*]` |
| `xdp_acl_deny_total` | Counter | (none) | `acl_deny` |
| `xdp_parse_errors_total` | Counter | (none) | `parse_errors` |
| `xdp_sampler_errors_total` | Counter | `kind` | sampler-internal |
| `xdp_attached_mode` | Gauge | `mode="native"\|"drv"\|"skb"` | from EBPF-2-04 |

Naming-convention nailed in `rel` cross-review ┬¦2 / EBPF-2-08:
**`action`** is the label key (not `result`). The mapping
`Action::* ŌåÆ label value` lives in `lb-observability` (not in
`lb-l4-xdp`) so kernel-side index changes don't break consumers.

Sampler shape identical to REL-2-12's ŌĆö single task, 1 s cadence.
One `init_xdp_sampler(loader, cadence)` call: each tick reads
`StatsSnapshot`, diffs against previous to update Counters, sets
Gauges directly, increments `xdp_sampler_errors_total` on read
failure (retains previous values, no panic).

Per-CPU summation happens **inside** `read_snapshot` (EBPF-2-08
hands already-summed values per agreed contract). If contract
flips to a per-CPU slice, rel sums here ŌĆö internal refactor.

Proof:
Extend `tests/xdp_metrics.rs` with:
1. `test_xdp_packets_total_by_action`:
   - Inject 100 packets across the XDP path on a synthetic netns;
     fan out 60 forwarded, 30 dropped, 10 passed_to_kernel via a
     test program that returns the corresponding `XDP_*` codes.
   - Scrape `/metrics`; assert each `xdp_packets_total{action}`
     reflects the right count.
2. `test_xdp_attached_mode_native_or_skb`:
   - Force SKB mode via the EBPF-2-04 probe-result hook; assert
     `xdp_attached_mode{mode="skb"} == 1` and other modes == 0.
3. `test_per_cpu_summation_no_double_count`:
   - Bind sampler to a 4-CPU netns; inject a single packet on CPU 2;
     after one tick, `xdp_packets_total{action="forwarded"}` increments
     by exactly 1.
4. `test_sampler_resets_baseline_on_loader_replace`:
   - Replace the `XdpLoader` (simulating reattach); next tick assigns
     a fresh baseline rather than producing negative deltas.

Risk / blast radius:
Shares the single XDP sampler task with REL-2-12. Adding metric
families adds prometheus series ŌĆö bounded cardinality (5 + 2 + 1 +
1 + small + 3 = ~12 series). Scrape cost trivial. If the
`StatsSnapshot` schema in EBPF-2-08 evolves, rel's `record_deltas`
needs an update; the test above catches missing fields via a
`#[non_exhaustive]` annotation on the struct.
