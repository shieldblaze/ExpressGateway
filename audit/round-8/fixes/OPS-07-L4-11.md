# Fix plan — OPS-07 + L4-11 (bundle): bpffs mount-type guard + systemd hardening

Owner: div-ops.
Findings folded in:
- ROUND8-OPS-07 (medium) — systemd unit lives only inside `DEPLOYMENT.md` as a code block; lacks `SystemCallFilter=@system-service`, `SystemCallArchitectures=native`, `RestrictAddressFamilies`, `RestrictNamespaces`, `ProtectKernel{Tunables,Modules,Logs}`, `ProtectControlGroups`, `ProtectProc=invisible`, `UMask=0077`; no CI gate against `systemd-analyze security`; no `WatchdogSec`.
- ROUND8-L4-11 (medium) — loader does not statfs-check that pin dir is `bpffs` (`BPF_FS_MAGIC = 0xCAFE4A11`); doc-comment lies about this. Container deployments without bind-mounted `/sys/fs/bpf` fail with opaque `EbpfError::Map(InvalidPin(...))`.

Coverage-gap themes cited:
- **Theme 1 — "Verified-Fixed" snapshot of script existence, not capability** (OPS-07). SEC-2-11 closed `CAP_SYS_ADMIN` and was graded `Verified-Fixed`; nobody ran `systemd-analyze security` against the documented unit. The grade snapshotted "capability bounded" not "unit hardened."
- **Theme 4 — Multi-validator audit handoff** (L4-11). EBPF-2-05 audited the pinning policy; SEC-2-11 audited the capability set; neither validator walked the *intersection* (bpffs mount-type at runtime).

The bundle: OPS-07 ships the unit as a file with `RequiresMountsFor=/sys/fs/bpf`; L4-11 catches the *missed* mount at runtime with a clear error. Belt + braces.

### A.2 — Watchdog wire-in

Implementation in `crates/lb/src/main.rs`: when `NOTIFY_SOCKET` env is set (set by systemd when `NotifyAccess=main`), spawn a `tokio::time::interval(5s)` task that calls `sd_notify::notify(false, &[NotifyState::Watchdog])`. Requires adding `sd-notify = "0.4"` to `crates/lb/Cargo.toml`. The task is gated on the env variable so it's a no-op outside systemd.

A separate sibling task that watches the runtime health (one of: tokio runtime metric `worker_alive_count`, an `/healthz`-style internal probe) — for v1 we just heartbeat unconditionally. Future work: tie the watchdog to actual liveness (e.g. accept-loop has serviced at least one accept in the last 10s).

### A.4 — CI gate

`.github/workflows/ci.yml` new job:
```yaml
systemd-analyze:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - run: sudo apt-get update && sudo apt-get install -y systemd-container
    - run: systemd-analyze --no-pager security packaging/expressgateway.service
    - run: |
        score=$(systemd-analyze --no-pager security packaging/expressgateway.service 2>&1 | tail -1 | awk '{print $NF}')
        # Score is "1.5" or similar — extract the numeric leading number.
        threshold=1.5
        awk -v s="$score" -v t="$threshold" 'BEGIN { exit !(s < t) }'
```

If `systemd-analyze` is not available on the GH-hosted runners' default image, the job uses `docker run --rm -v $PWD:/repo redhat/ubi9-init systemd-analyze ...` as the fallback (verified at plan-approval).

## B. L4-11 — `bpffs` mount-type guard

### B.1 — Runtime check in `XdpLoader::load_from_bytes_pinned`

In `crates/lb-l4-xdp/src/loader.rs:527-544`, BEFORE `loader.map_pin_path(p)`:
```rust
use nix::sys::statfs::{statfs, FsType};
const BPF_FS_MAGIC: i64 = 0xCAFE4A11;

let st = statfs(p).map_err(|e| XdpLoaderError::PinPathStatfs(p.to_path_buf(), e))?;
if st.filesystem_type().0 != BPF_FS_MAGIC {
    return Err(XdpLoaderError::PinPathNotBpffs {
        path: p.to_path_buf(),
        actual_magic: st.filesystem_type().0,
        hint: format!("run: mount -t bpf bpffs {}", p.display()),
    });
}
```

Add new error variants:
```rust
pub enum XdpLoaderError {
    // ...existing...
    PinPathStatfs(PathBuf, nix::Error),
    PinPathNotBpffs { path: PathBuf, actual_magic: i64, hint: String },
}
```

Implement `Display` so the hint is the operator-actionable error text.

### B.2 — Doc comment correction

`crates/lb-l4-xdp/src/loader.rs:58-62` doc-comment currently lies about runtime enforcement. Update to:
> The pin path must be a `bpffs` mountpoint (`mount -t bpf bpffs <path>`). Mount-type is verified at runtime in `load_from_bytes_pinned`; a non-bpffs path returns `XdpLoaderError::PinPathNotBpffs` with a hint that includes the canonical mount command. The systemd unit (`packaging/expressgateway.service`) declares `RequiresMountsFor=/sys/fs/bpf` so the unit fails to start if the mount is absent.

## C. Bundle interaction

OPS-07's `RequiresMountsFor=/sys/fs/bpf` is the *systemd-side* defense. L4-11's runtime statfs is the *process-side* defense. Both should land — OPS-07 alone fails if an operator runs the binary outside systemd; L4-11 alone allows an operator to start the unit on a host where `/sys/fs/bpf` was unmounted between unit-start and process-up.

## E. Risk / open questions

- `RestrictNamespaces=true` may conflict with operator use of `nsenter` for debugging. Document in the unit header.
- `ProtectClock=true` blocks `clock_settime`; we never call it. Confirmed.
- `ProcSubset=pid` may hide `/proc/cpuinfo` from the process; the `num_cpus` crate uses `sched_getaffinity` not `/proc/cpuinfo`, so this is fine. Confirmed.
- `nix::sys::statfs::FsType.0` field access: API is stable in nix ≥0.27. Verify the version in `Cargo.lock` at plan-approval.
- `BPF_FS_MAGIC` literal: kernel header `linux/magic.h` defines `BPF_FS_MAGIC 0xcafe4a11`; we duplicate the constant rather than depend on a kernel-header crate.
