Proof:
- The proof here is workflow run logs, not a cargo test. Acceptance
  criteria:
  - A PR that adds a `RUSTSEC-*` ignored crate causes audit job
    to fail (verified manually with a throwaway commit prior to
    merge ŌĆö the PR description must link to that test run).
  - A PR that adds an unused `[dependency]` causes machete to
    fail (same).
  - A PR that adds a new `unsafe fn` without bumping the baseline
    causes the unsafe-inventory job to fail (same).
- Static check (sec runs locally before opening PR):
  - `act -j audit -W .github/workflows/ci.yml` with a poisoned
    `Cargo.toml` reproduces failure.
  - For tests that *can* run in this repo's CI:
    `cargo test --test ci_smoke -- --ignored ci_workflow_yaml_parses`
    (a tiny new test in `tests/ci_smoke.rs` that parses the yaml and
    asserts `cargo audit --deny warnings` literal is present, and
    the geiger + machete jobs exist by name).
- Sanity assertion in the same test: `audit/security/baseline-unsafe-count.json`
  exists and parses as `{"total": <number>}`.

Risk / blast radius:
- CI gets slower (geiger compiles the whole tree under instrumentation
  ŌĆö ~3 min added). Mitigation: cache via `Swatinem/rust-cache@v2`;
  job is independent so it parallelizes.
- False positives in machete for crates with feature-gated re-exports
  (a known issue). Mitigation: `Cargo.toml` ignore-list with comments
  per crate.
- A new `RUSTSEC-*` on a transitive that we cannot upgrade quickly
  will block all PRs. **Documented escape valve**: add to
  `.cargo/audit.toml` ignore list with a justification line + an
  issue link. The quarterly review re-evaluates.
