# Plan for SEC-2-01 — Wire SmuggleDetector into H1/H2 hot path

Finding-ref:     SEC-2-01 (high, current status: Open)

Files touched:
- `crates/lb-l7/Cargo.toml` (add `lb-security` dep; sec owns)
- `crates/lb-l7/src/h1_proxy.rs` (call site; sec owns wiring per §D)
- `crates/lb-l7/src/h2_proxy.rs` (call site; sec owns wiring per §D)
- `crates/lb-l7/src/h2_to_h1.rs` (H2→H1 downgrade check site; sec owns)
- `crates/lb-security/src/smuggle.rs` (add strict TE-codec policy
  per SEC-2-15; sec owns)
- `crates/lb-l7/tests/smuggle_wired.rs` (new; sec owns)
- `crates/lb-l7/tests/h2_to_h1_pseudo_strip.rs` (new; sec owns)

Joint boundary:
- `proto` owns PROTO-2-10 (`:authority`/Host coherence) in
  `h2_proxy.rs`. Round-4 call ordering: SmuggleDetector first (400),
  authority check second (421), strip list third.
- `code` owns CODE-2-01 (dep-graph umbrella). Sec performs the
  `Cargo.toml` edit since the call sites are in lb-l7.

Approach:
Add `lb-security = { path = "../lb-security" }` to `lb-l7/Cargo.toml`.
In `H1Proxy::handle`, immediately after `req` is destructured into
parts but **before** the hop-by-hop strip and **before** upstream
acquire, call:
```
SmuggleDetector::check_all(parts.headers, SmuggleMode::H1)
    .map_err(|e| reply_400(e))?;
```
In `H2Proxy::handle`, call the same with `SmuggleMode::H2` to catch
CL+TE that arrive via HTTP/2 (rare but legal per RFC 9113 §8.1).
In `H2ToH1::translate` (the H2→H1 downgrade synth) call
`SmuggleDetector::check_h2_downgrade(&h2_headers)` before writing the
H1 request line; reject 400 on `Err(SecurityError::H2Downgrade)`.
Extend `SmuggleDetector` with strict TE-codec policy: reject any
`Transfer-Encoding` whose codec list contains anything beyond
`chunked` (per SEC-2-15 matrix). Add `SmuggleMode::H1Strict` opt-in
via `[runtime].strict_te = true`; default lenient.

Proof:
- `cargo test -p lb-l7 --test smuggle_wired` containing:
  - `test_cl_te_rejected` — POST with `Content-Length: 5` +
    `Transfer-Encoding: chunked` reaches `H1Proxy::handle`, asserts
    response is 400 and upstream socket count is unchanged.
  - `test_te_gzip_chunked_strict_rejected` — POST with
    `Transfer-Encoding: gzip, chunked` under strict_te=true returns
    400; under strict_te=false (default) passes through (regression
    guard so the knob is exercised both ways).
  - `test_duplicate_cl_same_value_rejected` — two identical
    Content-Length headers, expect 400 (sec's stricter-than-hyper
    behaviour per SEC-2-15).
- `cargo test -p lb-l7 --test h2_to_h1_pseudo_strip::test_h2_downgrade_pseudo_blocked`
  — H2 request with a forbidden `:authority`-bearing forged header
  reaches translate, asserts the synthesized H1 request line never
  enters the upstream socket buffer (assert via mock upstream).
- Invariant: every test asserts the request is rejected **before**
  `Upstream::acquire` is called (use a mock upstream that panics on
  acquire).

Risk / blast radius:
- Hot-path header scan (~8 lookups/req); bench target <2µs added
  latency. Fallback gate: `[runtime].smuggle_check = true` default.
- False-positive risk on unusual CL+TE clients — opt-out via
  `smuggle_check = false` per listener.
- Joint conflict in `h2_proxy.rs` (proto-owned): sec adds
  `pre_proxy_smuggle_check()` in `lb-security/src/smuggle.rs`;
  proto inserts the call inside PROTO-2-10. Sec provides API, proto
  inserts call site.
