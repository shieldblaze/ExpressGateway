# Plan for SEC-2-14 — Wire lb-compression or remove the crate

Finding-ref:     SEC-2-14 (medium, current status: Open)

  - `Cargo.toml` (workspace; remove member entry — code's file per

Joint boundary:
- This plan **proposes Option A** (remove) and requests lead
  arbitration. The workspace `Cargo.toml` edit belongs to code per
  §D; sec authors the decision rationale here and asks code to fold
  the removal into CODE-2-15. **Sec does not edit Cargo.toml**.
- If lead arbitrates Option B, the wiring edits in h{1,2}_proxy.rs

  `Decompressor` API extension only.

Approach (A — remove):
Recommended: (a) pass-through proxy doesn't decompress, (b) no
caller, (c) unused defence crate = false confidence + supply-chain
surface, (d) future filters can re-introduce.

Implementation:
1. CODE-2-15 removes `lb-compression` from workspace members.
2. SEC-2-07 cargo-machete catches dangling refs.
3. ADR `audit/security/adr/0001-lb-compression-removed.md` records
   rationale + reintroduction conditions (BREACH guard, gRPC
   max-message-size, JSON validation).

Approach (B — wire, only if lead overrides):
1. Add dep to `lb-l7/Cargo.toml`.
2. Extend `Decompressor::decompress` with global `Arc<Semaphore>`.
3. Compute `max_bytes = min(caller, mem_budget / max_concurrent)`
   at startup.
4. Wire into body-inspection filter chain — none exists today, so
   Option B is a feature, not a fix.

Proof (Option A — remove):
- `cargo build --workspace` succeeds after removal (proves no
  hidden consumer).
- `cargo test --workspace` succeeds (proves no test depends).
- New `tests/no_lb_compression.rs::test_crate_absent` asserts
  `lb_compression` is not in `cargo metadata --format-version 1`'s
  package list. Runs in CI; regression-protects the removal.
- The cargo-machete CI gate (SEC-2-07) verifies no dangling
  references.

Proof (Option B — wire):
- `cargo test -p lb-l7 --test decompress_bomb::test_10gib_bomb_rejected`
  — 10-byte gzip whose decompressed output is 10 GiB; assert
  decompression aborts at the global ceiling (≤256 MiB by default)
  with `Err(BombDetected)`; assert process RSS does not grow.
- `cargo test -p lb-compression --test global_ceiling::test_concurrent_decompress_bounded`
  — spawn `max_concurrent_decompress + 1` decompress tasks; assert
  the (+1) blocks on the semaphore and total resident memory
  ≤ `max_bytes × max_concurrent_decompress`.

Risk / blast radius (Option A):
- Risk: deletes a working crate. If a future feature needs it, the
  reintroduction is a few hours of work (the code remains in git
  history; restoration is `git revert`).
- No runtime risk — crate has zero callers.

Risk / blast radius (Option B):
- Adds a memory-budget computation that is sensitive to the
  per-process `[runtime].max_concurrent_connections` knob. If
  operators raise the conn cap without raising the memory budget,
  decompress can be starved. Documented in RUNBOOK.

Recommendation to lead: **Option A**.
