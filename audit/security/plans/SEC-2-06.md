  `/startupz` split.** Sec's middleware wraps all rel's routes.
  Shared config struct sec defines, rel consumes:
  ```
  pub struct AdminHttpConfig {
      pub bind: SocketAddr,
      pub unsafe_bind_public: bool,
      pub bearer_token: Option<SecretString>,
      pub allowed_cidrs: Vec<IpNet>,
  }
  ```
  rel's REL-2-04 plan binds this struct's fields to TOML keys; sec
  enforces them in `serve()`.

Approach:
1. **Bind guard**: in `serve()`, after parsing `bind`, refuse to
   start if `!bind.ip().is_loopback() && !cfg.unsafe_bind_public`.
   Error message names the override key. Default config now sets
   `bind = "127.0.0.1:9090"`.
2. **Bearer token**: optional `Authorization: Bearer <token>` header
   check using `subtle::ConstantTimeEq` for the comparison. If
   `bearer_token = None`, no auth (loopback-only is fine; combined
   with bind guard the default is safe).
3. **IP allowlist**: if non-empty, peer IP must match a CIDR. Applied
   before bearer check.
4. **Public bind requires both**: if `unsafe_bind_public = true` and
   neither `bearer_token` nor `allowed_cidrs` is set, refuse to
   start with a hard error (no foot-gun).
5. Document the recommended recipe in `RUNBOOK.md` (rel's file): mTLS
   sidecar on unix socket, or loopback + sidecar scrape, or
   `unsafe_bind_public + bearer + allowlist`.

Proof:
- `cargo test -p lb-observability --test admin_http_auth::test_default_bind_is_loopback`
  — `AdminHttpConfig::default().bind` is `127.0.0.1:9090`.
- `cargo test -p lb-observability --test admin_http_auth::test_public_bind_without_auth_refused`
  — `serve(AdminHttpConfig { bind: 0.0.0.0:9090, ..default })`
  returns `Err` immediately.
- `cargo test -p lb-observability --test admin_http_auth::test_bearer_token_required`
  — start with token=Some, send GET /metrics without header → 401;
  with correct header → 200; with wrong header → 401.
- `cargo test -p lb-observability --test admin_http_auth::test_constant_time_compare`
  — wrong-length and wrong-prefix tokens both reject (negative test
  for timing side-channel: we assert via subtle's constant-time
  primitive, not by wall-clock; relies on subtle's audit).
- `cargo test -p lb-observability --test admin_http_auth::test_cidr_allowlist_enforced`
  — non-allowlisted peer receives 403.
- Invariant: no route on admin_http returns 200 unless (loopback) OR
  (allowlisted AND token-correct).

Risk / blast radius:
- Breaking change for any operator who already binds `0.0.0.0` and
  scrapes anonymously. **Rollout note**: the bind guard is enforced
  by default. Operators who depend on the current behaviour must set
  `unsafe_bind_public = true` AND provide a token or allowlist —
  this is the desired behaviour. Document in CHANGELOG.
