- `code` owns `crates/lb/src/main.rs` per §D. Accept-loop edit at

  code inserts call.** API:
  ```
  pub struct ConnGate {
      per_listener: Arc<Semaphore>,
      per_ip: Arc<Mutex<LruCache<IpAddr, Arc<Semaphore>>>>,
      per_ip_cap: usize,
  }
  impl ConnGate {
      pub fn new(listener_cap: usize, per_ip_cap: usize) -> Self;
      pub async fn try_acquire(&self, peer: IpAddr)
          -> Result<OwnedPermitPair, GateError>;
  }
  ```
  `try_acquire` returns immediately on exhaustion
  (`ListenerFull` / `PerIpFull`); pair is RAII.
- code closes the socket on `Err` (no spawn) per CODE-2-05.
- `EMFILE` backoff loop is **CODE-2-06**, not sec.

Approach:
Implement `ConnGate` in `lb-security/src/conn_cap.rs`. Use
`tokio::sync::Semaphore::try_acquire_owned` for the listener cap
(no blocking — overflow returns immediately). For per-IP, maintain a
`lru::LruCache<IpAddr, Arc<Semaphore>>` sized to
`listener_cap / per_ip_cap` (eviction is correct because a
fully-released IP has no outstanding permits and can be GC'd).
Atomics on the semaphore use AcqRel/Acquire per SEC-2-16
recommendation (the gate counter is security-gating).

Defaults: per-listener cap = 10_000 (overridable via
`[runtime].max_concurrent_connections`); per-IP cap = 256
(overridable via `[runtime].max_concurrent_connections_per_ip`).
Match QUIC's existing `max_connections: 100_000` knob for
consistency.

Failure mode on overflow: respond TCP RST (drop without
`shutdown(Write)`). For TLS listeners, RST before ServerHello —
prevents amplification.

Proof:
- `cargo test -p lb-security --test conn_cap::test_listener_cap_enforced`
  — spawn 11_000 attempts at cap=10_000; assert 1_000 receive
  `Err(GateError::ListenerFull)` and `try_acquire` never blocks
  longer than 100µs.
- `cargo test -p lb-security --test conn_cap::test_per_ip_cap_enforced`
  — 300 attempts from one IP at per_ip_cap=256; assert 44
  rejected, all 256 successful permits release-roundtrip cleanly.
- `cargo test -p lb-security --test conn_cap::test_lru_evicts_idle_ips`
  — fill LRU to capacity, idle one IP, assert it is evicted and a
  new IP succeeds.
- `cargo test -p lb-l7 --test conn_cap_integration::test_socket_flood_rejected`
  — open 11_000 TCP connections to a test listener at cap=10_000;
  assert the 10_001st receives RST within 1 ms (no FD leak: assert
  `active_connections` gauge ≤ 10_000 throughout).
- Invariant: under flood, the listener's accepted-and-spawned count
  never exceeds the cap; the rejected count appears in metric
  `connections_rejected_total{reason}`.

Risk / blast radius:
- Risk: legitimate burst clients from one egress NAT (e.g. a corp
  CGNAT) all share one IP and trip per_ip_cap=256. Mitigation:
  per-listener override and per-IP allowlist
  `[security].trusted_cidrs` (out of scope this round; documented in
  RUNBOOK.md as a follow-up — recorded as SEC-2-99-A new finding).
- Boundary in `main.rs` with code's CODE-2-05/06 plans: sec's API is
  finalized in this plan; code's plan must reference this API by
  exact name. Round-4 merge: code lands first (per §D
  serialisation); sec's API is already in `lb-security` (separate
  file) so no edit conflict.
