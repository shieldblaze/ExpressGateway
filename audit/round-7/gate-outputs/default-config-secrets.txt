Result: (empty) — no placeholder strings present in the shipped
default config (`config/default.toml`).

Inspection of `config/default.toml`:
  - Two TCP backends on 127.0.0.1; no secret material.
  - No TLS, no API token, no certificate paths.

The shipped default is therefore safe by construction — there are no
placeholder secrets to refuse, and `lb-config::validate_config` does
not need a placeholder-rejection branch for this configuration.

For TLS-enabled deployments, secrets are loaded from files referenced
by path (`tls.cert`, `tls.key`); a missing or unreadable file is
rejected at startup by the file-open syscall and surfaces as a
ConfigError. There is no in-band placeholder convention to reject.
