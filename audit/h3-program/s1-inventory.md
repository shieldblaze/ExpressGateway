## quiche integration detail

- **Versions** (`Cargo.lock`): `quiche 0.28.0` (checksum `882f4a3c…`, BoringSSL backend — deps include `boring`, `qlog`, `octets`), `tokio-quiche 0.18.0` (checksum `cbc6bca0…`). Workspace pins in root `Cargo.toml:85-86` (`quiche = "0.28"`, `tokio-quiche = "0.18"`); `crates/lb-quic/Cargo.toml:17-18` consumes both via `{ workspace = true }`. Migration tracked in `docs/decisions/quinn-to-quiche-migration.md` (from quinn 0.11 + rustls/ring).
- **Server config** (`listener.rs:345-368` `build_server_config`, rebuilt per accepted connection because `quiche::Config` holds non-Send state post cert-load):
  - `quiche::Config::new(quiche::PROTOCOL_VERSION)`
  - `set_application_protos(H3_ALPN_PROTOS)` → ALPN `h3` (+ `h3-29`); proven by `tests/quic_alpn_h3.rs` (`server_advertises_h3`, `server_accepts_h3_29_legacy_client`, `server_rejects_unknown_alpn`, `production_alpn_constant_is_h3`).
  - Flow control: `initial_max_data 10MiB`, `bidi_local/bidi_remote/uni 1MiB`, `initial_max_streams_bidi 16`, `initial_max_streams_uni 16`; UDP payload `set_max_recv_udp_payload_size`/`set_max_send_udp_payload_size(1350)`; `set_max_idle_timeout`.
  - **TLS**: BoringSSL via `load_cert_chain_from_pem_file` + `load_priv_key_from_pem_file` (filesystem PEM paths). Client-side upstream verification (`verify_peer(true)` + CA) covered by `tests/h3_upstream_verify.rs` / `tests/proto_translation_e2e.rs`.
- **Stream API**: `conn_actor.rs::poll_h3` uses `conn.readable()` + `conn.stream_recv()`; responses via `StreamTx` + `conn.stream_send()`; `h3_bridge.rs` (h3→h3) drives `quiche::connect` upstream with `stream_send(sid, chunk, fin)` on client bidi stream 0.
- **Connection migration**: `set_disable_active_migration(true)` (`listener.rs:362`) — migration is explicitly DISABLED on the server config. No migration/path-validation pass-through (relevant to capability #4 PARTIAL).
- **0-RTT replay**: `lb_security::ZeroRttReplayGuard` is constructed and held by `QuicListener` (`listener.rs:186`) and threaded to the router (`RouterParams.replay_guard`). The guard's detection contract is unit-proven by `tests/quic_native.rs::replay_filter_rejects_second_0rtt_with_same_token`; that test's own comment states the quiche wire path "does not yet feed this filter per-Initial" via the `QuicEndpoint` helper — wire-level per-Initial replay feeding lives in the router path (Pillar 3b.3c). Status: guard wired into listener/router; end-to-end per-Initial wire enforcement through `QuicListener` is not proven by a crate-local test.
- **QUIC DATAGRAM**: quiche unreliable-datagram API is exercised only by the `QuicEndpoint` test helper (`roundtrip_datagram`, `tests/quic_native.rs`). It is NOT plumbed through the production `QuicListener`/router/actor path (no datagram handling in `conn_actor.rs`). Relevant to #4 and any future WebTransport/connect-udp work.
- **RETRY / stateless retry**: `RetryTokenSigner` constructed + held by `QuicListener` (`listener.rs:185`) and threaded into the router; router doc (`listener.rs:7-10`) states it handles "RETRY wire handling". Not exercised by a crate-local test (router accept loop uncovered in `cargo test -p lb-quic`).
