# RFC 9000 — QUIC Transport Protocol

## Summary

RFC 9000 defines QUIC: a UDP-based transport that provides ordered
byte streams, connection-level flow control, built-in TLS 1.3 via RFC
9001, connection migration, 0-RTT data, and a variable-length integer
encoding used throughout HTTP/3. ExpressGateway splits the QUIC surface
between two crates: `lb-quic`, which models the datagram/stream
forwarding surface that the proxy pipeline consumes, and `lb-h3`, which
owns the variable-length integer codec (RFC 9000 §16) that HTTP/3
framing depends on. Because running a real QUIC stack in CI is
intractable, `lb-quic` is intentionally minimal — the production path
will link Quinn (PROMPT.md §12) but the proxy's responsibility for
datagram validation, connection-ID routing, path validation, and
replay protection is represented by standalone units that can be
tested against the RFC's MUST clauses without a network.

## Scope in ExpressGateway

- `crates/lb-quic/src/lib.rs` — `QuicDatagram`, `QuicStream`,
  `forward_datagram:60`, `forward_stream:73`, and the `QuicError`
  enumeration (`lib.rs:19`).
- `crates/lb-h3/src/varint.rs` — QUIC variable-length integer codec.
  - `MAX_VARINT = (1 << 62) - 1` at `varint.rs:17`.
  - `decode_varint:27` — 2-bit prefix dispatch to 1/2/4/8-byte decodes.
  - `encode_varint:68` — minimal-length encoding per §16.
- `crates/lb-security/src/zero_rtt.rs` — `ZeroRttReplayGuard` for the
  anti-replay window required by RFC 9001 §9.2 and referenced by
  RFC 9000 §21.11.
- `tests/quic_native.rs` — datagram/stream forwarding regression.
- `tests/security_zero_rtt_replay.rs` — anti-replay regression.

## MUST clauses we implement

- **Variable-length integer encoding (§16)** — The 2-bit MSB prefix
  selects 1/2/4/8-byte widths (6/14/30/62 usable bits). `decode_varint`
  dispatches on `first >> 6` and returns the decoded value plus the
  byte count; `encode_varint` picks the shortest legal width and
  rejects values above `MAX_VARINT`. Section 16 of RFC 9000 requires
  this exact encoding.
- **Datagrams MUST be treated as atomic units (§12.1)** — QUIC carries
  multiple frames in a single UDP datagram; partial forwarding is
  forbidden. `forward_datagram` treats the `data` blob as one unit and
  either forwards the full copy or rejects with `EmptyPayload`.
- **Empty payload rejection** — A QUIC datagram with zero payload is
  not a legal QUIC packet (§12.4 requires at least one frame). Our
  `forward_datagram` rejects the degenerate case up front; regression
  in `tests/quic_native.rs`.
- **Connection ID routing (§5.1)** — `QuicDatagram.connection_id`
  carries the Destination Connection ID the endpoint uses to route the
  datagram to the correct stack instance. The routing layer in
  `lb-l7`/`lb-l4-xdp` keys on this field. The MUST here is that a
  datagram whose DCID does not match any active connection MUST either
  be discarded or processed as a stateless retry trigger; the proxy's
  routing layer implements the discard path.
- **Stream framing carries `fin` (§3.2)** — Every stream frame has an
  implicit "stream finished" bit; `QuicStream.fin` at `lib.rs:51`
  mirrors this. Per §3.2, once a stream enters "Data Sent" → "Data
  Recvd" the endpoint MUST NOT send further STREAM frames; our
  forwarding code preserves the flag so downstream enforcement can
  observe it.
- **0-RTT replay protection (§21.11 / RFC 9001 §9.2)** — Non-idempotent
  requests MUST NOT be processed from 0-RTT data without replay
  defence. `ZeroRttReplayGuard` maintains a fixed-capacity hash-set of
  recently seen tokens with ring-buffer eviction; the hash is a 32-byte
  digest so large tokens don't inflate memory (see
  `crates/lb-security/src/zero_rtt.rs`). Regression:
  `tests/security_zero_rtt_replay.rs`.
- **Stateless retry validation (§8.1.2)** — Retry tokens are
  authenticated with an AEAD; PROMPT.md §12 mandates AES-GCM with
  expiration enforcement. The production implementation lives in the
  Quinn integration and validates the token before any stream state is
  created.
- **Path validation on migration (§8.2)** — A client may migrate to a
  new IP/port mid-connection. The server MUST probe the new path with
  PATH_CHALLENGE/PATH_RESPONSE before committing to the new address.
  PROMPT.md §12 item 4 captures this requirement; Quinn performs the
  handshake.
- **Flow control (§4)** — Connection-level MAX_DATA and stream-level
  MAX_STREAM_DATA MUST be honoured; sender MUST NOT exceed the
  advertised credit. Handled inside Quinn; the proxy exposes it only
  through configured window sizes.
- **TLS 1.3 is mandatory (§4.2)** — QUIC carries only TLS 1.3 Initial
  and Handshake packets; TLS 1.2 is rejected at ALPN negotiation.
  PROMPT.md §12 captures "QUIC mandates TLS 1.3".
- **Packet numbers MUST be monotonically increasing per packet-number
  space (§12.3)** — Enforced inside Quinn; the proxy's stream forwarder
  never reorders frames because STREAM frames carry their own offsets.
- **Maximum varint value (§16)** — Implementations MUST NOT produce
  varint values greater than `2^62 - 1`. `encode_varint` rejects any
  value above `MAX_VARINT` with `H3Error::InvalidVarint`
  (`varint.rs:69`).

## Edge cases & security

- **0-RTT replay** — Attacker replays an early-data packet capturing a
  non-idempotent request. `ZeroRttReplayGuard` hashes the token and
  inserts into a bounded ring buffer so a duplicate within the capacity
  window is rejected (`SecurityError::ZeroRttReplay`). The hash keeps
  memory bounded even if a client sends many distinct but large tokens.
- **Amplification attack** — A QUIC server MUST NOT send more than
  three times the bytes it has received from an unverified client
  address (§8.1). Enforced by Quinn, surfaced by the proxy only
  indirectly via Retry.
- **Connection-ID flood** — An attacker issues many NEW_CONNECTION_ID
  frames to pressure the server's CID cache. Quinn caps the per-
  connection CID pool (default 8).
- **Stateless reset oracles** — `QuicError::InvalidConnectionId`
  surfaces mismatches without revealing internal state.
- **Packet-injection** — AEAD-protected headers make injection
  impractical; short packet numbers are expanded locally before AEAD
  verification.
- **Fuzzing** — The varint codec is a natural fuzz target; a 1/2/4/8-
  byte dispatch table is exercised by the `proptest` regressions in
  `hpack`-adjacent suites.

## Known deviations / TODO

- `lb-quic` is a **simulation surface**: the production wiring to
  Quinn (PROMPT.md §12) is planned but the CI-safe shim does not run
  an AEAD, emit packets, or negotiate congestion control. The
  forwarding code exists so the pipeline interfaces can be wired and
  regression-tested without sockets.
- Datagram extension (RFC 9221 "Unreliable Datagrams") is out of
  scope.
- Multipath QUIC (draft-ietf-quic-multipath) is not implemented.
- Connection migration is accepted by Quinn but path validation is the
  library's responsibility; the proxy does not expose migration
  telemetry.
- Spin bit and statistical probing are not surfaced.
- Session resumption tickets are issued by the TLS layer (RFC 9001);
  the proxy does not encrypt them separately.
